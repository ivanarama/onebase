# Быстрый старт onebase

## Что такое onebase

Открытая бизнес-платформа, похожая по концепции на 1С:Предприятие.  
Одна **информационная база** = метаданные (справочники, документы, регистры) + данные в PostgreSQL.  
Метаданные описываются в YAML-файлах, бизнес-логика — на простом DSL (похожем на 1С).

---

## Путь 1 — Через лаунчер (без командной строки)

Подходит для быстрого старта и разработки конфигурации через редактор файлов.

### Шаг 1. Скачать и запустить

1. Скачайте архив со страницы [Releases](https://github.com/ivantit66/onebase/releases/latest):  
   `onebase-windows-amd64.zip`
2. Распакуйте в любую папку, например `C:\onebase\`
3. Двойной клик на **`onebase-gui.exe`**

Откроется лаунчер информационных баз:

```
┌─────────────────────────────────────────┐
│  onebase — Информационные базы          │
│  [1С:Предприятие] [Обновить БД]         │
│  [+ Добавить]                        [✕]│
├──────────────────────┬──────────────────┤
│  Нет информационных  │                  │
│  баз                 │                  │
└──────────────────────┴──────────────────┘
```

### Шаг 2. Убедиться, что PostgreSQL запущен

Лаунчер создаст базу данных автоматически, но PostgreSQL должен быть установлен и запущен.  
Строка подключения по умолчанию: `postgres://postgres:пароль@localhost/имя_бд?sslmode=disable`

### Шаг 3. Создать новую базу

Нажмите **«+ Добавить»** и заполните форму:

| Поле | Пример |
|---|---|
| Наименование | Мой склад |
| Тип конфигурации | **Файловый (разработка)** |
| Путь к папке | `C:\мои-проекты\склад` |
| Строка подключения | `postgres://postgres:1234@localhost/склад_dev?sslmode=disable` |
| Порт | 8080 |
| ☑ Создать структуру папок | включить |

Нажмите **«Добавить»**.

**Что произойдёт автоматически:**
- Создастся база данных `склад_dev` в PostgreSQL
- Создастся папка `C:\мои-проекты\склад\` со структурой:

```
склад/
├── config/
│   └── app.yaml          ← имя приложения
├── catalogs/
│   └── контрагент.yaml   ← пример справочника
├── documents/
│   └── счёт.yaml         ← пример документа
├── registers/            ← регистры накопления
├── reports/              ← отчёты
└── src/
    └── счёт.os           ← пример бизнес-логики
```

### Шаг 4. Запустить базу

Выберите базу в списке → нажмите **«1С:Предприятие»**.  
Откроется веб-интерфейс в браузере: **http://localhost:8080/ui**

### Шаг 5. Редактировать конфигурацию

Откройте папку `C:\мои-проекты\склад\` в любом редакторе (VS Code, Notepad++).  
Добавляйте и изменяйте YAML-файлы — база перезагружает конфигурацию **автоматически** при каждом сохранении файла.

## Путь 2 — Через командную строку

Подходит для CI/CD, автоматизации и разработки без GUI.

### Требования

```
Go 1.22+       → https://go.dev/dl/
PostgreSQL 14+ → https://www.postgresql.org/download/
```

### Установка

```bash
git clone https://github.com/ivantit66/onebase
cd onebase
go build -o onebase.exe ./cmd/onebase
```

### Создать проект

```bash
onebase init --name "Мой склад" --project ./склад
cd склад
```

### Запустить в режиме разработки

```bash
onebase dev \
  --project . \
  --db "postgres://postgres:1234@localhost/склад_dev?sslmode=disable"
```

Открыть: **http://localhost:8080/ui**

## Структура конфигурации

### Справочник — `catalogs/товар.yaml`

```yaml
name: Товар
fields:
  - name: Наименование
    type: string
  - name: Единица
    type: string
  - name: Цена
    type: number
```

### Документ — `documents/приходная.yaml`

```yaml
name: Приходная
posting: true                    # включает кнопки «Провести» / «Отменить»
fields:
  - name: Номер
    type: string
  - name: Дата
    type: date
  - name: Поставщик
    type: reference:Контрагент   # ссылка на справочник
tableparts:
  - name: Товары
    fields:
      - name: Товар
        type: reference:Товар
      - name: Количество
        type: number
      - name: Цена
        type: number
      - name: Сумма
        type: number
```

### Регистр накопления — `registers/остатки.yaml`

```yaml
name: Остатки
dimensions:
  - name: Товар
    type: string
resources:
  - name: Количество
    type: number
```

### Бизнес-логика — `src/приходная.os`

Файл называется так же, как документ (без расширения).

```
Процедура ПриЗаписи()

  Если this.Номер = "" Тогда
    Error("Номер обязателен");
  КонецЕсли;

  // Автоматический подсчёт суммы строк
  Для Каждого Строка Из this.Товары Цикл
    Строка.Сумма = Строка.Количество * Строка.Цена;
  КонецЦикла;

  // Движения по регистру (применяются при проведении)
  Движения.Остатки.Очистить();
  Для Каждого Строка Из this.Товары Цикл
    Дв = Движения.Остатки.Добавить();
    Дв.ВидДвижения = "Приход";
    Дв.Товар = Строка.Товар;
    Дв.Количество = Строка.Количество;
  КонецЦикла;

КонецПроцедуры
```

## Типы полей

| Тип | Описание | Пример |
|---|---|---|
| `string` | Текст | `"ООО Ромашка"` |
| `number` | Число | `42.5` |
| `date` | Дата и время | `"2026-04-24"` |
| `bool` | Да / Нет | `true` |
| `reference:Имя` | Ссылка на справочник или документ | — |

---

## DSL — встроенные объекты

| Объект | Что даёт |
|---|---|
| `this` | текущий объект (поля документа или справочника) |
| `this.ТабЧасть` | коллекция строк табличной части |
| `Движения.ИмяРегистра` | движения по регистру накопления |
| `Error("текст")` | прервать запись, показать ошибку пользователю |

---

## Проведение документов

Если в YAML документа указано `posting: true`:

- В интерфейсе появляются кнопки **Провести** и **Отменить проведение**
- Движения по регистрам записываются **только при проведении**
- При отмене проведения движения удаляются

Без `posting: true` движения записываются сразу при сохранении.

## Конфигуратор

Кнопка **«Конфигуратор»** в лаунчере открывает инструмент разработчика — страницу платформы, не пользовательский режим.

### Вкладка «Дерево метаданных»

Раскрываемое дерево всей конфигурации:

- **Справочники** — список полей с типами
- **Документы** — поля, флаг «проводится», табличные части
- **Модули** — прямо под каждым объектом раскрывается исходный код `.os` файла
- **Регистры накопления** — измерения, ресурсы, реквизиты
- **Отчёты** — параметры и текст запроса

Работает в обоих режимах: файловом (читает с диска) и database (выгружает из `_onebase_config`).

### Вкладка «Конвертер 1С»

Позволяет импортировать конфигурацию из 1С:Предприятие.

**Как выгрузить конфигурацию из 1С:**
1. Открыть Конфигуратор 1С
2. Меню **Конфигурация → Выгрузить конфигурацию в файлы**
3. Указать папку

**Затем в конфигураторе onebase:**

| Кнопка | Что делает |
|---|---|
| **Просмотр** | Конвертирует во временную папку, показывает отчёт — сколько объектов переехало, что пропущено, предупреждения по типам |
| **Конвертировать и применить** | То же + сразу импортирует в базу |

Для database-режима — импортирует в `_onebase_config`. Для file-режима — копирует файлы в папку проекта.

### Вкладка «Файлы»

Для database-режима:
- **Выгрузить** — экспортирует файлы в `~/.onebase/workspace/<id>/` и открывает папку в проводнике
- **Загрузить** — импортирует из указанной папки обратно в базу данных

Для file-режима: файлы лежат на диске и редактируются напрямую в любом редакторе. Сервер перезагружает конфигурацию автоматически при изменении файлов.

## Команды CLI (справочно)

```bash
onebase start                          # лаунчер
onebase dev  --project . --db <dsn>    # режим разработки (hot reload)
onebase run  --project . --db <dsn>    # production-режим
onebase migrate --project . --db <dsn> # применить миграцию вручную
onebase ibases list                    # список баз в лаунчере
```
