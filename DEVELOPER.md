# Руководство разработчика onebase

## Структура проекта

```
my-project/
├── catalogs/       # YAML-описания справочников
├── documents/      # YAML-описания документов
├── registers/      # YAML-описания регистров накопления
├── reports/        # YAML-описания отчётов
└── src/            # DSL-скрипты событий (*.os)
```

Запуск сервера:
```bash
onebase dev --project ./my-project --db "postgres://localhost/mydb?sslmode=disable"
```

---

## Справочники (Catalogs)

Файл: `catalogs/<имя>.yaml`

```yaml
name: Номенклатура
fields:
  - name: Наименование
    type: string
  - name: Артикул
    type: string
  - name: Цена
    type: number
  - name: Активен
    type: bool
```

### Типы полей

| Тип          | PostgreSQL   | Описание                           |
|--------------|--------------|------------------------------------|
| `string`     | TEXT         | Строка                             |
| `number`     | NUMERIC      | Число                              |
| `date`       | TIMESTAMPTZ  | Дата и время                       |
| `bool`       | BOOLEAN      | Да/Нет                             |
| `reference:X`| UUID         | Ссылка на справочник или документ X|

Пример ссылочного поля:
```yaml
  - name: Контрагент
    type: reference:Контрагент
```

## Документы (Documents)

Файл: `documents/<имя>.yaml`

```yaml
name: Поступление
fields:
  - name: Номер
    type: string
  - name: Дата
    type: date
  - name: Поставщик
    type: reference:Контрагент
tableparts:
  - name: Товары
    fields:
      - name: Номенклатура
        type: string
      - name: Количество
        type: number
      - name: Цена
        type: number
      - name: Сумма
        type: number
```

Табличные части (`tableparts`) — вложенные списки строк внутри документа.

## Регистры накопления (Registers)

Файл: `registers/<имя>.yaml`

```yaml
name: ТоварноеДвижение
dimensions:
  - name: Номенклатура
    type: string
resources:
  - name: Количество
    type: number
  - name: Сумма
    type: number
attributes:
  - name: Комментарий
    type: string
```

- **dimensions** — разрезы учёта (по чему группируем остатки)
- **resources** — числовые показатели, которые накапливаются со знаком (Приход +, Расход −)
- **attributes** — дополнительные данные, не влияющие на итоги

Период движения заполняется **автоматически** из первого поля типа `date` документа-регистратора.

## DSL-скрипты

Файл: `src/<имя_документа>.os`

Скрипт называется так же, как документ (регистр первой буквы), расширение `.os`.

### Процедура ПриЗаписи / OnWrite

Вызывается при сохранении объекта. Если процедура вызывает `Error(...)` — сохранение отменяется, ошибка показывается пользователю.

```
Процедура ПриЗаписи()
  // Валидация
  Если this.Номер = "" Тогда
    Error("Номер обязателен");
  КонецЕсли;

  // Движения регистра
  Движения.ТоварноеДвижение.Очистить();
  Для Каждого Строка Из this.Товары Цикл
    Строка.Сумма = Строка.Количество * Строка.Цена;
    Дв = Движения.ТоварноеДвижение.Добавить();
    Дв.ВидДвижения = "Приход";       // или "Расход"
    Дв.Номенклатура = Строка.Номенклатура;
    Дв.Количество   = Строка.Количество;
    Дв.Сумма        = Строка.Сумма;
  КонецЦикла;
КонецПроцедуры
```

### Доступные конструкции

```
// Присваивание
x = выражение;

// Условие
Если условие Тогда
  ...
[Иначе
  ...]
КонецЕсли;

// Цикл по коллекции
Для Каждого Строка Из this.ТабличнаяЧасть Цикл
  ...
КонецЦикла;

// Арифметика: + - * /
// Сравнения:  = <> > < >= <=

// Доступ к полям документа
this.ИмяПоля

// Движения регистра
Движения.НазваниеРегистра.Очистить()     // удалить старые строки
Дв = Движения.НазваниеРегистра.Добавить()  // добавить строку
Дв.ИмяИзмерения = значение
Дв.ВидДвижения  = "Приход"  // или "Расход"
```

### Встроенные функции

| Функция       | Описание                                              |
|---------------|-------------------------------------------------------|
| `Error(текст)`| Отменить сохранение, показать ошибку пользователю     |

### Билингвальность

Все ключевые слова поддерживаются на **русском и английском** одновременно:

| Русский         | English        |
|-----------------|----------------|
| Процедура       | Procedure      |
| КонецПроцедуры  | EndProcedure   |
| Если            | If             |
| Тогда           | Then           |
| Иначе           | Else           |
| КонецЕсли       | EndIf          |
| Для Каждого Из  | For Each In    |
| Цикл            | Do             |
| КонецЦикла      | EndDo          |

## Отчёты (Reports)

Файл: `reports/<имя>.yaml`

```yaml
name: ОстаткиТоваров
title: Остатки товаров
params:
  - name: НаДату
    type: date
    label: На дату
  - name: ВидДвижения
    type: select
    label: Вид движения
    options:
      - ""
      - Приход
      - Расход
query: |
  ВЫБРАТЬ
    Номенклатура,
    СУММА(ВЫБОР КОГДА вид_движения = "Приход" ТОГДА количество ИНАЧЕ -количество КОНЕЦ) КАК Количество
  ИЗ РегистрНакопления.ТоварноеДвижение
  ГДЕ (&НаДату ЕСТЬ ПУСТО ИЛИ period <= &НаДату)
  СГРУППИРОВАТЬ ПО Номенклатура
  УПОРЯДОЧИТЬ ПО Номенклатура
```

### Типы параметров

| Тип      | Описание                     |
|----------|------------------------------|
| `string` | Текстовое поле               |
| `number` | Числовое поле                |
| `date`   | Поле выбора даты             |
| `select` | Выпадающий список (`options`)|

Пустой параметр передаётся как NULL. Используй `ЕСТЬ ПУСТО` для опциональной фильтрации:
```sql
ГДЕ (&Параметр ЕСТЬ ПУСТО ИЛИ поле = &Параметр)
```

### Язык запросов

Поддерживаются русские и английские ключевые слова:

```sql
ВЫБРАТЬ [РАЗЛИЧНЫЕ]
  поле [КАК Псевдоним],
  СУММА(поле) КАК Итог,
  ВЫБОР КОГДА условие ТОГДА значение ИНАЧЕ другое КОНЕЦ КАК Поле
ИЗ РегистрНакопления.НазваниеРегистра
   | Справочник.НазваниеСправочника
   | Документ.НазваниеДокумента
[ГДЕ условие]
[СГРУППИРОВАТЬ ПО поле, ...]
[УПОРЯДОЧИТЬ ПО поле [УБЫВ|ВОЗР], ...]
```

**Агрегатные функции:** `СУММА`, `КОЛИЧЕСТВО`, `МИНИМУМ`, `МАКСИМУМ`, `СРЕДНЕЕ`  
**Логика:** `И`, `ИЛИ`, `НЕ`, `ЕСТЬ ПУСТО`  
**Параметры:** `&ИмяПараметра` — подставляется как позиционный аргумент `$N`

## Миграция базы данных

При запуске `dev` или `run` миграция применяется автоматически (`CREATE TABLE IF NOT EXISTS` + `ALTER TABLE ADD COLUMN IF NOT EXISTS`).

Явный запуск:
```bash
onebase migrate --project ./my-project --db "postgres://localhost/mydb?sslmode=disable"
```

Удаление и переименование полей **не поддерживается** автоматически — делать вручную через SQL.
