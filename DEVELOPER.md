# Руководство разработчика onebase

## Структура проекта (пользовательское приложение)

```
my-project/
├── config/
│   └── app.yaml          # имя и версия приложения
├── catalogs/             # YAML-описания справочников
├── documents/            # YAML-описания документов
├── registers/            # YAML-описания регистров накопления
├── reports/              # YAML-описания отчётов
└── src/                  # DSL-скрипты событий (*.os)
```

Запуск в файловом режиме (hot reload):
```bash
onebase dev --project ./my-project --db "postgres://localhost/mydb?sslmode=disable"
```

---

## Справочники (Catalogs)

Файл: `catalogs/<имя>.yaml`

```yaml
name: Номенклатура
fields:
  - name: Наименование
    type: string
  - name: Артикул
    type: string
  - name: Цена
    type: number
  - name: Активен
    type: bool
```

### Типы полей

| Тип | PostgreSQL | Описание |
|---|---|---|
| `string` | TEXT | Строка |
| `number` | NUMERIC | Число |
| `date` | TIMESTAMPTZ | Дата и время |
| `bool` | BOOLEAN | Да/Нет |
| `reference:X` | UUID | Ссылка на справочник или документ X |

```yaml
  - name: Контрагент
    type: reference:Контрагент
```

## Документы (Documents)

Файл: `documents/<имя>.yaml`

```yaml
name: Поступление
fields:
  - name: Номер
    type: string
  - name: Дата
    type: date
  - name: Поставщик
    type: reference:Контрагент
tableparts:
  - name: Товары
    fields:
      - name: Номенклатура
        type: string
      - name: Количество
        type: number
      - name: Цена
        type: number
      - name: Сумма
        type: number
```

Табличные части (`tableparts`) — вложенные списки строк внутри документа.

## Регистры накопления (Registers)

Файл: `registers/<имя>.yaml`

```yaml
name: ТоварноеДвижение
dimensions:
  - name: Номенклатура
    type: string
resources:
  - name: Количество
    type: number
  - name: Сумма
    type: number
attributes:
  - name: Комментарий
    type: string
```

- **dimensions** — разрезы учёта
- **resources** — числовые показатели (Приход +, Расход −)
- **attributes** — дополнительные данные, не влияющие на итоги

Период движения берётся автоматически из первого поля типа `date` документа-регистратора.

## DSL-скрипты

Файл: `src/<имя_документа>.os`

Имя файла совпадает с именем документа (первая буква любого регистра), расширение `.os`.

### Процедура ПриЗаписи / OnWrite

Вызывается при сохранении. `Error(...)` отменяет сохранение.

```
Процедура ПриЗаписи()
  // Валидация
  Если this.Номер = "" Тогда
    Error("Номер обязателен");
  КонецЕсли;

  // Движения регистра
  Движения.ТоварноеДвижение.Очистить();
  Для Каждого Строка Из this.Товары Цикл
    Строка.Сумма = Строка.Количество * Строка.Цена;
    Дв = Движения.ТоварноеДвижение.Добавить();
    Дв.ВидДвижения  = "Приход";
    Дв.Номенклатура = Строка.Номенклатура;
    Дв.Количество   = Строка.Количество;
    Дв.Сумма        = Строка.Сумма;
  КонецЦикла;
КонецПроцедуры
```

### Доступные конструкции

```
// Присваивание
x = выражение;

// Условие
Если условие Тогда
  ...
[Иначе
  ...]
КонецЕсли;

// Ранний выход / возврат значения
Возврат;
Возврат выражение;

// Цикл по коллекции
Для Каждого Строка Из this.ТабличнаяЧасть Цикл
  ...
КонецЦикла;

// Числовой цикл
Для i = 1 По 10 Цикл
  ...
КонецЦикла;

// Арифметика: + - * /
// Сравнения:  = <> > < >= <=

// Поля документа
this.ИмяПоля

// Накопительная переменная
ИтогСумма = 0;
Для Каждого Строка Из this.Товары Цикл
  ИтогСумма = ИтогСумма + Строка.Сумма;
КонецЦикла;

// Движения регистра
Движения.НазваниеРегистра.Очистить()
Дв = Движения.НазваниеРегистра.Добавить()
Дв.ВидДвижения = "Приход"   // или "Расход"
```

### Встроенные функции

**Ошибки:** `Error(текст)` — отменить сохранение.

**Даты:** `Today()` / `ТекущаяДата()`, `Now()` / `ТекущаяДатаВремя()`, `Year(d)` / `Год(d)`, `Month(d)` / `Месяц(d)`, `Day(d)` / `День(d)`

**Строки:** `Str(n)` / `Строка(n)`, `Number(s)` / `Число(s)`, `Upper(s)` / `ВРег(s)`, `Lower(s)` / `НРег(s)`, `TrimAll(s)` / `СокрЛП(s)`, `Left(s,n)` / `Лев(s,n)`, `Right(s,n)` / `Прав(s,n)`, `Mid(s,start,len)` / `Сред(s,start,len)`, `StrLen(s)` / `СтрДлина(s)`, `StrFind(s,sub)` / `СтрНайти(s,sub)`

**Математика:** `Round(n,d)` / `Окр(n,d)`, `Abs(n)` / `Абс(n)`, `Int(n)` / `Цел(n)`, `Max(a,b)` / `Макс(a,b)`, `Min(a,b)` / `Мин(a,b)`

### Билингвальность

Все ключевые слова работают на русском и английском одновременно:

| Русский | English |
|---|---|
| Процедура | Procedure |
| КонецПроцедуры | EndProcedure |
| Если / Тогда / Иначе / КонецЕсли | If / Then / Else / EndIf |
| Для Каждого Из / Цикл / КонецЦикла | For Each In / Do / EndDo |

## Отчёты (Reports)

Файл: `reports/<имя>.yaml`

```yaml
name: ОстаткиТоваров
title: Остатки товаров
params:
  - name: НаДату
    type: date
    label: На дату
  - name: ВидДвижения
    type: select
    label: Вид движения
    options:
      - ""
      - Приход
      - Расход
query: |
  ВЫБРАТЬ
    Номенклатура,
    СУММА(ВЫБОР КОГДА вид_движения = "Приход" ТОГДА количество ИНАЧЕ -количество КОНЕЦ) КАК Количество
  ИЗ РегистрНакопления.ТоварноеДвижение
  ГДЕ (&НаДату ЕСТЬ ПУСТО ИЛИ period <= &НаДату)
  СГРУППИРОВАТЬ ПО Номенклатура
  УПОРЯДОЧИТЬ ПО Номенклатура
```

### Типы параметров

| Тип | Описание |
|---|---|
| `string` | Текстовое поле |
| `number` | Числовое поле |
| `date` | Поле выбора даты |
| `select` | Выпадающий список (`options`) |

Пустой параметр → NULL. Для опциональной фильтрации:
```
ГДЕ (&Параметр ЕСТЬ ПУСТО ИЛИ поле = &Параметр)
```

### Язык запросов

```
ВЫБРАТЬ [РАЗЛИЧНЫЕ]
  поле [КАК Псевдоним],
  СУММА(поле) КАК Итог,
  ВЫБОР КОГДА условие ТОГДА значение ИНАЧЕ другое КОНЕЦ КАК Поле
ИЗ РегистрНакопления.НазваниеРегистра
   | Справочник.НазваниеСправочника
   | Документ.НазваниеДокумента
[ГДЕ условие]
[СГРУППИРОВАТЬ ПО поле, ...]
[УПОРЯДОЧИТЬ ПО поле [УБЫВ|ВОЗР], ...]
```

**Агрегатные функции:** `СУММА`, `КОЛИЧЕСТВО`, `МИНИМУМ`, `МАКСИМУМ`, `СРЕДНЕЕ`  
**Логика:** `И`, `ИЛИ`, `НЕ`, `ЕСТЬ ПУСТО`  
**Параметры:** `&ИмяПараметра`

## Лаунчер информационных баз

`onebase start` открывает лаунчер — аналог «Запуск 1С:Предприятие».

### Регистр баз: `~/.onebase/ibases.yaml`

```yaml
bases:
  - id: <uuid>
    name: Склад
    config_source: database   # или: file
    path: ""                  # только для file-режима
    db: postgres://localhost/sklad?sslmode=disable
    port: 8080
    created: 2026-04-24T00:00:00Z
```

### Два режима конфигурации

**`config_source: file`** — конфигурация в папке на диске:
- Удобно для разработки, git-friendly
- `onebase dev --project <путь>` — hot reload при изменении файлов
- Путь к папке виден в лаунчере

**`config_source: database`** — конфигурация в PostgreSQL (`_onebase_config`):
- Используется для готовых баз у пользователей
- Создаётся из лаунчера («Добавить» → галочка «Создать с нуля»)
- Редактирование: кнопка **«Конфигуратор»** → «Выгрузить» → правка → «Загрузить»
- Выгрузка попадает в `~/.onebase/workspace/<base-id>/`

### Служебные таблицы

Создаются автоматически при первом запуске базы:

| Таблица | Назначение |
|---|---|
| `_onebase_config` | Конфигурация в БД-режиме |
| `_users` | Пользователи |
| `_sessions` | Сессии авторизации |

## Пользователи и авторизация

По умолчанию (пустая таблица `_users`) — доступ без пароля.

Как только создан хотя бы один пользователь — при входе требуется логин и пароль.

### Создание пользователей

В веб-интерфейсе базы: **Администрирование → Пользователи → + Добавить**.

Поля: логин, полное имя, пароль, флаг «Администратор».

### Роли

- **Администратор** (`is_admin = true`) — полный доступ + управление пользователями
- **Обычный пользователь** — доступ к данным базы

### Авторизация через лаунчер

При нажатии «1С:Предприятие» в лаунчере — если база защищена, перед открытием появится форма входа. После успешного входа открывается веб-интерфейс уже с активной сессией.

### API-эндпоинты авторизации

| Метод | URL | Описание |
|---|---|---|
| GET | `/login` | Страница входа |
| POST | `/login` | Форма входа |
| POST | `/logout` | Выход |
| GET | `/auth/status` | `{"requires_auth": bool}` |
| POST | `/auth/login` | Вход JSON (для лаунчера) |
| GET | `/auth/bootstrap?token=...` | Передать сессию в новое окно |

## Миграция базы данных

При запуске `dev` или `run` миграция применяется автоматически  
(`CREATE TABLE IF NOT EXISTS` + `ALTER TABLE ADD COLUMN IF NOT EXISTS`).

Явный запуск:
```bash
onebase migrate --project ./my-project --db "postgres://localhost/mydb?sslmode=disable"

# В режиме конфигурации в БД:
onebase migrate --config-source database --db "postgres://localhost/mydb?sslmode=disable"
```

Удаление и переименование полей **не поддерживается** автоматически — вручную через SQL.

---

## Флаги команд `dev` / `run` / `migrate`

| Флаг | По умолчанию | Описание |
|---|---|---|
| `--project` | `.` | Путь к папке конфигурации (для `file`-режима) |
| `--db` | `$DATABASE_URL` или `postgres://localhost/onebase` | Строка подключения к PostgreSQL |
| `--port` | `8080` | Порт HTTP-сервера |
| `--config-source` | `file` | Источник конфигурации: `file` или `database` |

## Структура платформы (для контрибьюторов)

```
cmd/onebase/          — точка входа
internal/
  api/                — HTTP-сервер, роутинг
  auth/               — пользователи, сессии, middleware
  cli/                — Cobra-команды (dev, run, migrate, start, ibases, init)
  configdb/           — хранение конфигурации в PostgreSQL
  devserver/          — файловый вотчер для hot reload
  dsl/                — лексер, парсер, AST, интерпретатор
  launcher/           — лаунчер: store, runner, HTTP-сервер, шаблоны, окно
  metadata/           — загрузка и валидация YAML-метаданных
  project/            — загрузка проекта (из папки и из БД)
  query/              — транслятор языка запросов → SQL
  report/             — загрузка отчётов
  runtime/            — реестр объектов, движения регистров
  storage/            — PostgreSQL: CRUD, DDL, миграция
  ui/                 — веб-интерфейс: шаблоны, хэндлеры, админка
examples/
  simple-erp/         — пример: склад с поступлением и списанием
```

### Сборка

```bash
# Обычная (без CGo, без нативного окна)
CGO_ENABLED=0 go build -o onebase ./cmd/onebase

# GUI с нативным окном WebView2 (Windows, требует CGo и MSVC)
go build -tags webview -ldflags="-s -w -H windowsgui" -o onebase-gui.exe ./cmd/onebase
```

### Тесты

```bash
# Юнит-тесты (без PostgreSQL)
go test ./...

# Интеграционные тесты (требуют PostgreSQL)
TEST_DATABASE_URL="postgres://localhost/onebase_test?sslmode=disable" \
  go test -tags=integration ./...
```

Интеграционные тесты покрывают:
- `integration_test.go` — сквозной сценарий в file- и database-режимах
- `internal/auth/integration_test.go` — Create/Authenticate/Middleware с реальной БД
- `internal/configdb/repo_test.go` — Export/Import round-trip
